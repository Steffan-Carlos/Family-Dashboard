# Projects Registry

> Each project tag groups related tasks into a development phase. Use phases to sequence feature rollouts.

| Tag | Name | Phase | Status | Description |
|-----|------|-------|--------|-------------|
| project-scaffold | Project Scaffolding | 1 | active | Initial repo setup, package.json, directory structure, dev configs |
| auth | Authentication | 1 | planned | PIN-based family member authentication, session management |
| calendar | Calendar System | 2 | planned | iCal feed parsing, calendar UI, event display |
| chores | Chore Tracking | 2 | planned | Chore assignment, completion tracking, streaks, rewards |
| dashboard | Dashboard UI | 3 | planned | Main dashboard layout, navigation, kiosk mode shell |
| settings | Settings & Admin | 3 | planned | Family member management, calendar feed config, app settings |

## Phase Guide

- **Phase 1**: Foundation - get the app running with basic auth
- **Phase 2**: Core features - calendar and chore systems
- **Phase 3**: Polish - dashboard shell, settings, kiosk mode

## Status Values

- `active` - Currently being worked on
- `planned` - Scoped but not started
- `paused` - Started but deprioritized
- `complete` - All tasks done, feature shipped
